## 第一步：创建 Supabase 数据库（约 4 分钟）

### 1.1 注册并创建项目

1. 打开 https://supabase.com ，点击 **Start your project**，用 GitHub 或邮箱注册
2. 点击 **New Project** 创建新项目
   - **Name**：随便填，如 `hupping-system`
   - **Database Password**：设一个密码（记好，后面要用）
   - **Region**：选离你最近的（如 Southeast Asia (Singapore)）
3. 等待约 2 分钟，项目创建完成

### 1.2 执行数据库脚本

1. 进入项目后，左侧菜单点击 **SQL Editor**
2. 点击 **New query**
3. 打开本部署包中的 `setup.sql` 文件，**复制全部内容**
4. 粘贴到 SQL Editor 中，点击 **Run** 按钮
5. 看到 `Success. No rows returned.` 表示执行成功

### 1.3 获取连接信息（重要！）

1. 左侧菜单点击 **Project Settings**（齿轮图标）
2. 点击 **API**
3. 找到以下两项，**复制保存好**：

```
Project URL:    https://xxxxxxx.supabase.co     ← 这就是 SUPABASE_URL
anon public:    eyJhbGciOiJIUzI1NiIsInR5...     ← 这就是 SUPABASE_ANON_KEY
```

> ⚠️ anon key 是公开密钥，设计上就是给前端用的，不是私密密钥，放心使用。

## 第二步：配置 config.js（约 1 分钟）

用记事本或任何文本编辑器打开 `config.js`，修改前两行：

```js
// ❌ 修改前
const SUPABASE_URL = 'https://YOUR-PROJECT.supabase.co';
const SUPABASE_ANON_KEY = 'YOUR-ANON-KEY';

// ✅ 修改后（填入你刚才获取的真实值）
const SUPABASE_URL = 'https://xxxxxxx.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsInR5...你的完整key';
```

**可选：修改班级口令**（在同一文件中找到）：

```js
const CLASS_PASSWORDS = {
  class1: '商数一班2024',   ← 改成你想要的口令
  class2: '商数二班2024'    ← 改成你想要的口令
};
```

保存文件。

---

## 第三步：部署到 Netlify（约 3 分钟）

### 3.1 注册 Netlify

1. 打开 https://app.netlify.com ，点击 **Sign up**，用邮箱/GitHub/Google 注册

### 3.2 拖拽部署

1. 登录后进入首页，找到 **Add new site** → **Deploy manually**
   （或直接访问 https://app.netlify.com/drop）
2. 将**整个 `deploy-static` 文件夹**拖拽到页面中央的虚线框区域
3. 等待几秒，部署完成！

### 3.3 获取永久链接

部署成功后，Netlify 会给你一个链接，格式类似：

```
https://hupping-system-xxxxxx.netlify.app
```

**这就是你的永久公网链接！** 发给学生即可使用。

### 3.4（可选）自定义域名

1. 点击站点 → **Domain settings** → **Add custom domain**
2. 输入你自己的域名（需提前购买域名）

## 使用说明

### 学生使用流程

1. 打开链接 → 选择班级 → 输入口令
2. 左侧选择自己所在的小组
3. 右侧依次点击其他小组进行评分（5 个维度滑块）
4. 提交评分后自动保存到云端
5. 切换「🏆 实时排名」查看排行榜

### 管理员操作

- **查看排名**：切换到排名页，点击任意小组查看维度详情
- **重置数据**：点击右上角「🔄 重置数据」按钮（需已验证口令）
- **修改口令**：编辑 `config.js` 中的 `CLASS_PASSWORDS`，重新拖拽部署

## 常见问题（FAQ）

**Q: 部署后学生访问报错怎么办？**
A: 检查 `config.js` 中的 SUPABASE_URL 和 KEY 是否填写正确，重新拖拽部署。

**Q: Supabase 一周没访问会暂停，数据会丢吗？**
A: 不会丢！数据永久保存。只需登录 Supabase 点击「Restore」恢复即可。互评期间每天都有人访问，不会触发暂停。

**Q: 可以同时两个班一起用吗？**
A: 可以！两个班数据相互独立，各自口令进入，互不影响。

**Q: 评分能修改吗？**
A: 能。已评分的小组点击后可重新打分，再次提交会覆盖之前的评分。

**Q: 需要付费吗？**
A: 不需要。Supabase 免费额度 500MB（本项目用量不到 1MB），Netlify 免费额度 100GB 带宽/月（本项目每月用量不到 1GB），永久免费。

**Q: 想修改班级/分组/评分维度怎么办？**
A: 编辑 `config.js` 中的 `CLASS_DATA`（分组数据）和 `DIMENSIONS`（评分维度），重新拖拽部署到 Netlify 即可。已有评分数据不受影响。
